Жобаның жұмыс атауы: **Task IN академиялық платформасын әзірлеу**.

### 2) Ақпараттық жабдықтау
Жобаға қажетті теориялық және практикалық база ретінде:
- білім беру веб-платформалары,
- клиент-сервер архитектурасы,
- рөлге негізделген қауіпсіздік (RBAC),
- REST API,

- Файл сақтау: `MinIO` (`minio` SDK, presigned URL)
- Қосымша инфрақұрылым: `nodemailer`

### 4) Техникалық құралдар
- Жеке компьютер (кемінде 8 ГБ RAM ұсынылады)
- SSD/HDD
- ОЖ: Windows 10/11 немесе macOS/Linux
- Интернет (package registry, сыртқы сервистер)
- Браузер: Chrome/Edge/Firefox
- Жергілікті немесе серверлік PostgreSQL, қажет болса Elasticsearch және MinIO

## Аңдатпа

### Қазақ тілінде
Бұл дипломдық жобада университеттік академиялық жұмыстарды орталықтандырылған түрде жинақтау, іздеу, жүктеу, рецензиялау және әкімшілендіру процестерін автоматтандыратын **Task IN** веб-платформасы қарастырылады. Жобаның өзектілігі білім беру ұйымдарында диплом, курстық және ғылыми жұмыстардың сапалы цифрлық айналымын ұйымдастыру қажеттілігімен түсіндіріледі.

Жүйе үш негізгі рөлге (студент, жетекші, әкімші) негізделген. Студенттер жұмыстарын жүктейді және мәртебесін бақылайды, жетекшілер бағалайды және кері байланыс береді, ал әкімші пайдаланушыларды және контент ағынын модерациялайды. Техникалық тұрғыдан жоба Next.js, TypeScript, Prisma, PostgreSQL, NextAuth, Elasticsearch және MinIO құралдарын пайдалана отырып жүзеге асырылған.

Нәтижесінде академиялық ортада қолдануға бейімделген толық стек платформа прототипі алынды.

**Кілт сөздер:** академиялық жұмыстар, веб платформа, Next.js, Prisma, PostgreSQL, Elasticsearch, MinIO, рөлдік модель.

### Орыс тілінде
В работе рассматривается разработка веб-платформы **Task IN** для управления академическими работами (дипломные, курсовые, статьи, проекты). Система поддерживает загрузку, модерацию, рецензирование, поиск и аналитику с учетом ролевой модели.

Реализованы роли студент/руководитель/администратор, защищенные маршруты, API для работы с контентом, хранение файлов в MinIO, поиск через Elasticsearch с резервным вариантом на PostgreSQL.

Результатом является работоспособный прототип платформы для образовательной среды.

### Ағылшын тілінде
This diploma project presents **Task IN**, a web platform for academic work lifecycle management (submission, moderation, review, search, and statistics). The system is role-based (student, supervisor, admin) and focuses on practical deployment in university workflows.

The implementation uses Next.js, TypeScript, Prisma, PostgreSQL, NextAuth, MinIO, and Elasticsearch (with SQL fallback). The final outcome is a functional full-stack prototype suitable for institutional adaptation.

## Кіріспе

Қазіргі жоғары білім беру жүйесінде академиялық жұмыстарды басқару процесі көп жағдайда шашыраңқы сервис пен қолмен орындалатын операцияларға тәуелді. Бұл деректердің жүйесіз сақталуына, верификация процесінің баяулауына және пайдаланушылар арасындағы өзара әрекеттесудің күрделенуіне әкеледі.

Осы мәселені шешу үшін Task IN жобасы бірыңғай цифрлық орта ұсынады: жұмысты жүктеуден бастап рецензия мен бекітуге дейінгі барлық кезеңдер бір веб интерфейсте жүргізіледі. Жоба архитектурасы қауіпсіздік, масштабталу және қолжетімділік қағидаларына сүйеніп құрастырылған.

- Материалдардың әртүрлі форматта және әртүрлі сақтау орындарында болуы
- Рөлдер мен рұқсаттардың бірыңғай ережесінің болмауы
- Жұмыс қабылдау/қайтару процестерінің ашық бақыланбауы
- Метадеректердің толық болмауы және іздеудің әлсіздігі

### 1.2 Пайдаланушы интерфейсі мен тәжірибе
Платформа интерфейсі әр рөлдің нақты міндетіне сай болуы тиіс: студент үшін жүктеу және бақылау қарапайым, жетекші үшін рецензиялау тиімді, әкімші үшін модерация жылдам болуы қажет. UX-тің негізгі талаптары: түсініктілік, жылдам навигация, қателерді нақты көрсету, responsive бейімделу.

4. UI интеграциясы  
5. Тестілеу және жетілдіру  
6. Қолданысқа енгізу

## 2. Task IN платформасын жобалау

### 2.1 Функционалдық талаптар
- Тіркелу және кіру (студент, жетекші, әкімші)
- Жетекші тіркеуін әкімші бекіту механизмі
- Академиялық жұмысты файлмен бірге жүктеу
- Жұмыс мәртебесін басқару (`PENDING`, `APPROVED`, `RETURNED`, `REJECTED`)
- Рецензия беру және пайдалы деп белгілеу
- Бетбелгі, жүктеу, хабарламалар, статистика
- Қоғамдық беттерде шектеулі ашық ақпарат

### 2.2 Функционалдық емес талаптар
- Қауіпсіздік: қорғалған маршруттар, пароль хэштелуі
- Өнімділік: іздеудің тиімділігі, сұраныс жылдамдығы
- Сенімділік: деректер тұтастығы, транзакциялық операциялар
- Масштабталу: MinIO және Elasticsearch қолдану мүмкіндігі
- Қолдану ыңғайы: адаптивті интерфейс

### 2.3 Технологиялар стегін таңдау негіздемесі
- **Next.js 14**: бір кодбазада UI және API
- **Prisma + PostgreSQL**: қатаң схема, қатынастарды басқару
- **NextAuth**: сенімді сессия және аутентификация
- **Elasticsearch**: кеңейтілген толықмәтіндік іздеу
- **MinIO**: файл сақтау мен жеткізуді икемді ұйымдастыру
- **Tailwind CSS**: жылдам және бірізді UI

### 2.4 Архитектура және функционалдық құрылым
Жүйе үш қабаттан тұрады:
1. Презентациялық қабат (`app`, `components`)
2. Қызметтік/бизнес қабат (`lib`, `middleware`, `route handlers`)
3. Деректер қабаты (`PostgreSQL`, `MinIO`, `Elasticsearch`)

Негізгі домен объектілері: `User`, `Work`, `Review`, `Faculty`, `Department`, `Keyword`, `Notification`, `Bookmark`, `Download`, `SiteSettings`.

## 3. Жүзеге асыру және жүйелік талдау

### 3.1 Іске асыру кезеңдері
- Жоба skeleton және конфигурация
- Prisma схемасы мен seed
- Аутентификация және рөлдік рұқсаттар
- Жұмыс жүктеу/модерация модульдері
- Іздеу және статистика
- Админ/жетекші панельдері

### 3.2 Дерекқор және API өзара әрекеттесуі
REST endpoint-тер `app/api/**` ішінде модульденген түрде құрылған:
- `auth` модулі
- `works` модулі
- `search` модулі
- `student/supervisor/admin` аймақтық модульдері
- `notifications`, `stats`, `profile` және т.б.

Middleware деңгейінде API мен беттерге рөлдік саясат қолданылады. Elasticsearch қолжетімсіз болса, жүйе автоматты түрде SQL іздеуге ауысады.

### 3.3 Навигация және қолжетімділік
Платформада:
- ашық (`/`, `/auth`) және қорғалған аймақтар бар;
- әр рөлге жеке маршруттар бөлінген;
- адаптивті layout десктоп және мобильді құрылғыларға бейімделген.

### 3.4 Ерекшеліктері және ұқсас платформалармен салыстыру
Task IN-нің басты ерекшеліктері:
- Рөлдік workflow-дың айқын моделі (студент-жетекші-әкімші)
- Академиялық процесс үшін арнайы сценарийлер
- Іздеу, файл сақтау және модерацияның бір платформада бірігуі
- Университеттік ортаға бейімдеуге ыңғайлы архитектура

## 4. Еңбекті қорғау және өміртіршілік қауіпсіздігі

### 4.1 Жалпы түсінік
Бағдарламашының жұмысы ұзақ уақыт монитор алдында отырумен байланысты болғандықтан, эргономика және еңбек гигиенасы талаптары сақталуы тиіс.

### 4.2 Жұмыс орнындағы техника қауіпсіздігі
- Монитор 50-70 см қашықтықта
- Үстел мен орындық биіктігі дұрыс реттелген
- Әр 60 минут сайын 5-10 минут үзіліс

### 4.3 Өрт қауіпсіздігі
- Электр сымдарының бүтіндігі
- Құрылғының қызып кетпеуін қадағалау
- Өрт сөндіру құралдарына қолжетімділік

### 4.4 Жарықтандыру
- Табиғи және жасанды жарықтың теңгерімі
- Экранға тікелей шағылысуды болдырмау

### 4.5 Электр қауіпсіздігі
- Сапалы адаптерлер мен желі фильтрлерін қолдану
- Ылғалды ортадан сақтау

### 4.6 Электромагниттік өрістер
Құрылғыларды қауіпсіз қашықтықта пайдалану және жұмыс уақытын реттеу ұсынылады.

### 4.7 Өндірістік микроклимат
20-24 C температура, 40-60% ылғалдылық, тұрақты желдету.
